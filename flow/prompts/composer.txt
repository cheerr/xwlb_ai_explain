## 二、处理流程（严格顺序）

### 1. 片头
- 截取 full.mp4 前 N 秒 (N = intro_TTS音频时长 + 2s缓冲)，保证口播说完不截断
- 去CCTV水印 → 竖屏背景+前景 → 叠加标题日期 → **有字幕**（开场白需要字幕展示）
- TTS音频替换原音轨，从 t=0 开始

### 2. 新闻段落（每条新闻执行）
- 找到对应切片视频（`{idx:02d}_*.mp4`），**idx必须使用CCTV原始分段序号（01-16），不能重编号**
- 切片前3秒自动裁剪（跳过主持人开场/转场），用 `ffmpeg -ss 3`
- 去CCTV水印（boxblur 4区域）
- 同时运行 TTS + SentenceBoundary 提取（见第三节）
- 竖屏合成：背景+前景->标题日期->全局关键词->字幕（用SentenceBoundary时间戳）
- **片头不叠加关键词**，仅新闻段叠加
- 关键词为整期节目统一提取（<=5字x3个），所有新闻段显示相同关键词
- "关键字："标签与关键词同色同大小(26px金加粗)
- 最后叠加TTS音频

### 2.1 news_index 映射规则 ⚠️
- 脚本JSON中的 `news_index` 必须是CCTV原始分段序号（01-16），不是0-based顺序编号
- 若LLM生成的脚本重编号了，需通过narration前20字匹配回原始索引
- 匹配方法：取narration前20字去编号 -> 查selected中的title -> 取index字段
- 匹配失败则该段视为无切片，fallback到黑底

### 3. 片尾 —— 投资总结（LLM生成+审核）

内容由独立的 `outro_summary` Agent 节点生成：
- 从已验证分析中提炼3-5条综合性投资建议
- 每条 <=40字，附带 <=30字原文证据
- 交叉结论优先（如"储能+AI双重驱动"）
- 措辞: "关注...""建议关注...""...值得留意""...存在机会"
- 禁止推荐具体股票/代码

布局:
- 背景：纯黑 `#000000`
- 标题：34px白 "投资总结"，**纵向居中**
- 建议：22px白，带序号，<=40字
- 证据：17px灰 `* 原文引用`，<=30字
- 底部：分隔线 + 15px灰风险提示
- 片尾视觉时长 = TTS音频时长 + 3s阅读缓冲
- 过渡效果：fade in 0.5s

### 4. 合并
- `ffmpeg -f concat -safe 0 -c copy` 拼接所有片段
- 叠加水印PNG：**必须** `-c:a copy -map 0:a:0`，禁止 `-an`
- 最终编码：`-c:v libx264 -preset fast -crf 22`

## 三、字幕系统——专业影视字幕规范 ⚠️

你是拥有10年经验的中文字幕师。以下每条规则来自真实制作经验，违反任何一条都会导致字幕不合格。

### 3.1 核心原则

| 原则 | 规则 | 原因 |
|------|------|------|
| 单行字数 | ≤25个汉字宽度 | 竖屏720px，22px字体，25字刚好撑满一行不换行 |
| 同时显示 | 屏幕同一时间只有1行字幕 | 观众视线固定，堆叠多行分散注意力 |
| 阅读节奏 | 每行停留 ≥2秒 | 中文阅读速度3-5字/秒，22字需4-7秒消化 |
| 语义完整 | 不在词语/数字/英文中间断句 | 打断认知单元造成理解障碍 |
| 同步精度 | 用TTS SentenceBoundary，禁止估算 | 字数比例与实际语速偏差可达40% |

### 3.2 宽度计算（数字/英文=0.5汉字）

```python
def char_width(text):
    """汉字=1, ASCII数字/字母/连接符=0.5, 其他=1"""
    w = 0
    for ch in text:
        if '一' <= ch <= '鿿' or '　' <= ch <= '〿':  # CJK字符
            w += 1
        elif ch.isascii() and (ch.isdigit() or ch.isalpha() or ch in '.-+%'):
            w += 0.5  # 数字/英文 + 小数点/负号/加号/百分号
        else:
            w += 1
    return w
```

`-` `+` `%` `.` 跟随数字/英文算0.5宽度，保证 "24.9%" "-3.5%" "AI+" 作为整体不拆断。

### 3.3 时间轴获取（唯一方式：SentenceBoundary）

```python
async def get_boundaries(text, audio_path):
    communicate = edge_tts.Communicate(text, "zh-CN-YunxiNeural", rate="+10%")
    boundaries = []
    async for chunk in communicate.stream():
        if chunk["type"] == "SentenceBoundary":
            s = chunk["offset"] / 10_000_000.0   # 100ns → 秒
            d = chunk["duration"] / 10_000_000.0
            boundaries.append({"text": chunk["text"].strip(), "start": s, "end": s+d})
    # 重新创建Communicate保存音频（stream已消费完毕）
    communicate2 = edge_tts.Communicate(text, "zh-CN-YunxiNeural", rate="+10%")
    await communicate2.save(str(audio_path))
    return boundaries
```

### 3.4 时间轴铁律

- ❌ **禁止 scale 缩放** — 缩放会导致相邻边界重叠，触发错误合并
- ❌ **禁止字数比例估算** — TTS语速不均匀，"储能是2026年"和"一二三四五六七八"同为7字但时长差3倍
- ❌ **禁止静态字幕** — 全段显示一行字幕不动，观众早已读完，字幕应随语音推进
- ✅ 时间戳原值直接用于 `enable='between(t,{start:.4f},{end:.4f})'`
- ✅ 最后一句 `end` 延伸到视频结束，避免最后几秒空白

### 3.5 碎片合并（< 0.5秒的边界并入前一句）

```python
merged = []
for b in cleaned:
    if merged and (b["end"] - b["start"]) < 0.5:
        merged[-1]["text"] += b["text"]
        merged[-1]["end"] = b["end"]
    else:
        merged.append(b.copy())
```

短于0.5秒的字幕人眼来不及阅读，必须合并。

### 3.6 编号保留

口播稿的序号 `1.` `2.` `3.` 在字幕中**正常保留**，不删除。
"1. 我国发电装机..." --字幕显示-> "1. 我国发电装机..."

### 3.7 长句拆分（> 22宽度时按标点断句，依次显示）

**拆分层级（优先级从高到低）：**
1. 逗号/分号 `，， ；；`  —— 最佳断点，天然语义停顿
2. 句号/感叹/问号 `。。！！？？`  —— 句子边界（**不含英文`.`, 防误拆`37.3` `24.9%`**）
3. 顿号 `、、`  —— 并列项之间
4. 词间（中文字符后，数字/英文序列前，或反之）—— 不得已才用

**硬性规则（违反即不合格）：**
- 数字序列不拆断：`37.3万户` `2500亿元` `-3.5%` `24.9%` `1.8亿千瓦` 必须完整
- 英文序列不拆断：`AI应用` `ETF基金` `A股市场` 必须完整
- 小数点 `.` 负号 `-` 加号 `+` 百分号 `%` 紧随数字，宽0.5，不计入断句符
- 拆分后每段按字数比例分配原时间区间
- 拆分后依次显示，不堆叠

```python
def split_long_subtitle(text, start_t, end_t):
    """长句按标点自然断句，逐行依次显示，分段后去标点"""
    if char_width(text) <= 25:
        return [{"text": clean_subtitle_text(text), "start": start_t, "end": end_t}]

    # 先按标点自然分段（优先在逗号处断）
    segments = _split_by_punctuation(text)

    # 对仍超宽的段，逐字累积拆分（保证数字/英文完整）
    chunks = []
    for seg in segments:
        if char_width(seg) <= 25:
            chunks.append(seg)
        else:
            current = ""
            for ch in seg:
                if char_width(current + ch) > 25 and current:
                    chunks.append(clean_subtitle_text(current))
                    current = ch
                else:
                    current += ch
            if current:
                chunks.append(clean_subtitle_text(current))

    # 去掉空字符串
    chunks = [c for c in chunks if c]
    if not chunks:
        return [{"text": clean_subtitle_text(text), "start": start_t, "end": end_t}]

    # 按字数比例分配时间（阅读速度由字数决定，不是宽度比例）
    char_counts = [len(c) for c in chunks]
    total_chars = sum(char_counts)
    total_dur = end_t - start_t

    result = []
    t = start_t
    for chunk, cc in zip(chunks, char_counts):
        dur = total_dur * (cc / total_chars) if total_chars > 0 else total_dur / len(chunks)
        result.append({"text": chunk, "start": t, "end": t + dur})
        t += dur
    return result
```

### 3.8 样式与清理
- 位置：前景视频画面内底部，距底48px，水平居中
- 字体：PingFang 22px，白色，1px黑阴影(70%透明)，不加粗
- 无背景条、无黑底、无边框
- **字幕文本首尾不得包含任何标点符号**
- 每个 boundary text 和 split chunk 都必须调 `text.strip('.!?！？。；;：:,，、')` 做**首尾双向清除**
- 从 SentenceBoundary 拿到原始文本后立即 strip，merge 后再次 strip，split 后每个 chunk 再次 strip
- sequenced 最终输出前再跑一次 `_strip_punct` 做强制清理

## 六、常见错误速查

| ❌ 症状 | ✅ 修复 |
|---------|---------|
| 视频无声 | 水印/字幕步骤去掉了 `-an`，加 `-c:a copy -map 0:a:0` |
| 字幕全并成1句 | 去掉了 scale 缩放，只合并<0.5s碎片 |
| 编号 `1.` `2.` 被删除了 | 不要删编号，字幕中正常保留 `1.` `2.` `3.` |
| 字幕首尾有逗号/句号 | `_strip_punct(text)` = `text.strip('.!?！？。；;：:,，、')` → 每个chunk必调 |
| 片头无字幕 | intro 的 `with_subtitles=True`，开场白需要字幕 |
| 字幕与语音不同步 | 用SentenceBoundary时间戳，不用字数估算 |
| 标题重复叠加 | 从clean base重新生成，不增量修补 |
| 前导静音 | intro TTS从t=0开始 |
| 口播与画面不对应 | news_index必须=CCTV原始序号(01-16)，不能重编号为0,1,2 |
| 主持人画面太多 | 切片 `ffmpeg -ss 3` 跳过前3秒转场 |
